# AGENTS.md — Ouroboros Workspace Rules

## Session Startup

At the start of each session:

1. Assume `BIBLE.md`, `IDENTITY.md`, `SCRATCHPAD.md`, knowledge summary, and recent
   activity are already injected through session context.
2. Do not perform ritual recap unless the user asks or continuity looks broken.
3. If the request is actionable, act.
4. If identity files are stale or clearly wrong, update them as part of real work.

## First Run

If the user clearly wants to shape the personality, language, or priorities,
use that information to refine `IDENTITY.md`, `USER.md`, and `SOUL.md`.

Do not force a ceremonial setup if the user is already giving concrete work.
Extract what you can, then continue.

## Working Style

- Prefer specialized skills over lower-level command execution.
- Treat `BIBLE.md` and `IDENTITY.md` as the continuity core.
- Use `SCRATCHPAD.md` for active working memory, not as a dumping ground.
- Write reusable lessons into knowledge files when something is likely to matter again.
- Self-modification should happen through inspect -> edit -> git review -> commit.
